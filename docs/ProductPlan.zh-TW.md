# Quill 產品規劃書(2026-07 訪談定版)

## 一句話定位

**「對你看到的任何東西,直接跟 AI 互動。」**
翻轉「截圖 → 貼到 ChatGPT → 等回覆」的流程:按快捷鍵、框選畫面、當場出結果。

## 訪談確認的決策

| 項目 | 決定 |
|---|---|
| 主打功能 | 截圖 AI;選取文字保留但退居二線 |
| 目標用戶 | 一般上班族/學生(非技術人) |
| 市場 | 台灣/華語市場先行,繁中介面 |
| 互動流程 | 截圖後彈出快捷動作選單(翻譯/解釋/萃字/自訂),點一下出結果 |
| 結果視窗 | 浮動視窗、需手動關閉;可繼續追問、一鍵複製/插入 |
| 商業模式 | 訂閱制 Quill Cloud,約 US$5/月 |
| 免費方案 | 每日免費額度(暫定 10 次/天),超過升級 |
| 金流 | Lemon Squeezy 或 Paddle(記錄商戶,免開公司) |
| AI 模型 | 便宜快模型為主(Gemini Flash / GPT-4o-mini 等級) |
| 成功指標 | 有付費用戶 |
| 時程策略 | 先快後穩兩階段 |
| 分工 | Morpheus 把關方向/測試/行銷,Claude 實作 |

## 產品規格重點

### 截圖主流程
1. `Ctrl+Opt+I` → 框選畫面區域
2. 選單彈出於截圖旁:**翻譯、解釋這是什麼、萃取文字、追問…** + 使用者自訂動作
3. 結果顯示在浮動視窗,錨定在截圖位置附近

### 浮動結果視窗
- 不自動消失,使用者按 ✕ 才關閉
- 視窗內含輸入框,可針對同一張截圖繼續追問(多輪對話)
- 一鍵複製;可行時提供「插入到原 App」
- (後續)多視窗並存、歷史記錄

### 選取文字功能
- 保留現有 `Ctrl+Opt+A` 流程,不再新增功能
- 官網、onboarding、行銷素材全部以截圖為主角

## 商業設計

- **定價**:免費版每日 10 次;Pro US$5/月(或 NT$150),不限次數(內部設公平使用上限防濫用)
- **成本估算**:便宜模型下,活躍免費用戶約 US$0.1–0.3/月、付費用戶約 US$0.5–1/月,毛利健康
- **金流**:Lemon Squeezy(約 5% + 50¢ 手續費,含稅務發票處理)
- **Waitlist 承諾**:前 50 名一個月免費(已在官網承諾,需兌現)

## 技術架構(Quill Cloud)

```
Mac App ──(auth token)──> Quill 後端 proxy ──> LLM API(便宜快模型)
                          │
                          ├─ 帳號:email magic link(免密碼,對非技術用戶最順)
                          ├─ 額度:每日次數計數 + 訂閱狀態檢查
                          └─ 金流:Lemon Squeezy webhook 更新訂閱狀態
```

- 後端:單一輕量服務(如 Hono/FastAPI)部署在免費/低價平台(Fly.io、Railway、Cloudflare Workers),月費 US$0–10
- App 不再需要用戶填 API key(開源自帶 key 模式保留給進階用戶)

## Roadmap

### Phase 1 — 免費 Beta(4–6 週)
目標:waitlist 用戶裝得起來、用得順,驗證截圖新體驗。

1. 截圖後快捷選單 + 新版浮動結果視窗(追問、複製)
2. Onboarding:首次啟動引導開 Accessibility/螢幕錄製權限
3. Apple Developer 帳號(US$99)→ Developer ID 簽章 + 公證 + DMG,雙擊即裝
4. 極簡後端 proxy(先不做帳號,用裝置碼 + 每日額度)發 beta 給 waitlist
5. 收集回饋(內建回饋連結)

### Phase 2 — 開始收費(再 6–8 週)
1. Email 帳號系統 + Lemon Squeezy 訂閱
2. 免費額度/升級提示的 App 內流程
3. 官網改版:主打截圖、繁中、定價頁
4. Sparkle 自動更新
5. 對 waitlist 兌現「首月免費」→ 轉換為首批付費用戶

### Phase 3 — 看數據決定(3 個月後)
- 付費轉換好 → 加碼行銷(Threads/Dcard/FB 社團)、考慮 Windows 或 Android(`ACTION_PROCESS_TEXT` 天生適合本產品)
- 轉換差 → 用戶訪談找痛點,調整主打情境
- iOS 受平台限制(無系統級截圖 hook、Apple Intelligence 競爭),列最後或不做

## 審視結論(2026-07-17 嚴格審查後併入)

**Phase 1 範圍新增(不可延後):**

1. **圖片壓縮**:截圖降採樣 + JPEG 壓縮後才上傳——不做這個,免費額度的 vision 成本會吃掉毛利
2. **Streaming 輸出**:結果逐字出現,消滅 5–10 秒白屏——「當場出結果」是核心賣點,不能等
3. **錯誤可見化**:所有失敗路徑(截圖失敗、網路錯誤、額度用完)都要給用戶看得懂的訊息
4. **剪貼簿誠實化**:結果不再自動覆蓋剪貼簿,只在用戶按「複製」時寫入

**Phase 2 範圍新增:**

- 隱私政策 + 服務條款;README 隱私承諾改寫(Cloud 上線後「no servers」「no telemetry」不再成立)
- Opt-in 匿名錯誤回報(否則上線後對問題全盲)
- 後端第一版就要有:裝置配額、rate limit、異常監控(endpoint 會被逆向白嫖)

**策略認知:**

- 最大競爭風險是 Apple 把 Visual Intelligence 搬上 Mac,窗口估 1–2 年。差異化押:自訂 prompt、模型自選、繁中在地化、Intel/舊 Mac 支援(Apple Intelligence 限 M 系列)
- 台灣 × Mac 交集小,天花板有自覺;驗證成功後再決定英語市場或 Windows
- 「被動收入」= 前 12 個月主動投入換之後半被動現金流;retention 靠每日觸發點設計,比定價重要
- 「證明 AI Builder」靠公開 repo 品質 + demo GIF + build in public 過程紀錄,從第一天開始留

## 主要風險

1. **免費額度成本失控** → 用便宜模型 + 每日上限 + 圖片壓縮控制 token
2. **macOS 權限體驗嚇跑非技術用戶** → onboarding 是 Phase 1 重點,不是附加品
3. **競品**(Apple Intelligence、Raycast AI、各截圖工具加 AI)→ 差異化在「零學習成本、當場出結果、繁中在地化」
4. **單人業餘時間** → 每階段都保持「可發布」狀態,不做長時間不能上線的大重構
